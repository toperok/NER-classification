# Оптимизация аналитической платформы через Multi-Task NLP

Проект по созданию единой Multi-Task NLP модели, которая объединяет распознавание именованных сущностей (**NER**) и **классификацию событий** в единый пайплайн. Это решение оптимизирует аналитическую платформу, снижает инфраструктурные затраты, устраняет дублирование данных и повышает точность извлечения ключевых фактов из новостного потока.

## Описание задачи

Задача: реализовать и обучить единую Multi-Task модель, которая одновременно решает две задачи: выделяет сущности в тексте и определяет категорию события. Это позволяет обрабатывать новостной поток в один проход вместо использования двух раздельных нейросетвых контуров.

Обучение и валидация модели проводятся на базе русскоязычного датасета **NEREL**, доступного на Hugging Face: [danasone/nerel](https://huggingface.co/datasets/danasone/nerel).

### Этапы работы:

1. **Исследовательский анализ (EDA)**: Изучение корреляций классов, плотности данных, распределения длин текстов и дисбаланса для выбора стратегии обучения.
2. **Реализация пайплайна**: Создание модульной Multi-Task архитектуры с общим энкодером и раздельными головами классификации.
3. **Обучение**: Запуск тренировочного цикла с дифференциальным Learning Rate, динамическим взвешиванием потерь и логированием метрик.
4. **Валидация и оптимизация**: Тестирование Joint-модели, проведение INT8-квантизации для CPU-инференса и анализ сложных примеров разметки.

---

# Метрики качества и результаты обучения

Для оценки качества работы Multi-Task модели использовались две специфичные конфигурации метрик:

* **Token F1-Macro Score (по всем сущностям)**: оценивает точность распознавания именованных сущностей (NER) на уровне отдельных токенов. Выбор Macro-усреднения позволяет одинаково учитывать качество распознавания как частых, так и редких типов сущностей, независимо от их количества в датасете.
* **Classification F1-Micro Score (по всем классам)**: измеряет общее качество категоризации событий новостного потока. Выбор Micro-усреднения объединяет глобальные показатели истинно/ложно положительных результатов по всем 30 классам, что дает объективную оценку общей точности платформы в условиях сильного дисбаланса.

### Результаты обучения по эпохам

Модель успешно достигла целевых порогов и нащупала оптимальный баланс для Joint-задачи на **15-й эпохе**, зафиксировав пиковые метрики качества:

### Эффективность стратегии оптимизации:
* **Совместное обучение**: Две головы на базе пулинга `Mean+Max` и динамического взвешивания потерь обеспечили стабильное качество распознавания на финальных этапах.
* **Дифференциальный Learning Rate**: Использование `2e-5` для тела RuBERT и `1e-4` для голов в сочетании с исключением весов лосса из Weight Decay позволило подтянуть отстающую NER-задачу без разрушения предобученных признаков базовой модели.
* **Стабилизация**: Планировщик `ReduceLROnPlateau` и Weight Decay для классификаторов успешно удержали Val Loss на минимальном плато ($\approx1.01$) вплоть до 12–15 эпох, минимизировав эффект взрывного переобучения.

# Оптимизация инференса и квантизация

Для снижения инфраструктурных затрат и ускорения работы платформы на CPU была проведена пост-тренировочная INT8-квантизация линейных слоев модели.

* **Производительность на CPU**: Скорость обработки токенов увеличилась в **1.70 раз**. Время инференса батча снизилось с 6.5374 сек до 3.8385 сек.
* **Плотность весов**: Эффективный размер модели уменьшился в **4 раза** (с $\approx680$ МБ до $\approx170$ МБ) благодаря сжатию весов каждого линейного слоя до 8 бит.
* **Стабильность разметки**: Квантизация полностью сохранила исходную точность генерации токенов. Распределение тегов NER и ранжирование топ-классов CLS остались идентичны полной версии.

**Итог**: Оптимизация обеспечила прирост скорости обработки последовательностей токенов на CPU без потери обобщающей способности Joint-архитектуры.

### Анализ «тяжелых» примеров (Ограничения модели):
Существует проблема в определении некоторых токенов, в особенности тех, что являются продолжением других. Например, в словосочетании *«муниципальных депутатов Москвы»* токен *«Москвы»* был ошибочно определен моделью как `I-PROFESSION`, тогда как правильный тег — `B-CITY`.

## Выводы по EDA и обоснование решений

1. **Общий Encoder**: Матрица корреляции показала, что задачи NER и классификации имеют общую структуру (например, сильная корреляция между `PENALIZED_AS` и `CONVICTED_OF`). Единый энкодер позволяет модели быстрее и дешевле формировать представления о каждой новости.
2. **Емкость модели**: Средняя плотность данных составляет $\approx12$, а на один документ приходится около 50 сущностей. Данные насыщены, поэтому выбрана емкая модель `DeepPavlov/rubert-base-cased` (регистрозависимая `cased`-версия критична для маркеров NER).
3. **Длина контекста**: Большинство текстов имеют длину от 150 до 300 токенов, что идеально укладывается в стандартный лимит RuBERT в 512 токенов без потери важной информации при обрезке.
4. **Борьба с дисбалансом**: Наблюдается дисбаланс классов (частые события встречаются в 10 раз чаще редких). Для стабилизации обучения используется взвешенная функция потерь (Weighted Loss).
